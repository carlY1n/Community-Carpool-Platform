# 拼车系统性能测试项目总览

## 📋 项目概述

本项目是基于Locust框架开发的拼车系统性能测试解决方案，专门针对您的UniApp拼车系统进行全面的API性能测试和负载测试。

## 🎯 项目目标

- **全面测试**: 覆盖认证、行程、预订等核心业务模块
- **真实模拟**: 模拟乘客、司机、管理员等真实用户行为
- **多场景支持**: 从轻负载到压力测试的完整场景覆盖
- **详细报告**: 提供HTML、CSV、JSON等多种格式的测试报告
- **易于使用**: 提供图形界面和命令行两种使用方式

## 📁 项目结构详解

## 🧪 测试场景说明

### 1. 认证模块测试 (auth_scenario.py)
- **AuthTestUser**: 基础认证功能测试
  - 登录性能测试
  - 注册性能测试
  - 登出测试
  - Token刷新测试

- **ConcurrentLoginUser**: 并发登录测试
  - 大量用户同时登录
  - 响应时间监控

- **InvalidAuthUser**: 异常认证测试
  - 无效凭据测试
  - 错误输入验证

- **AuthStressUser**: 认证压力测试
  - 快速API调用
  - 系统极限测试

### 2. 行程模块测试 (trip_scenario.py)
- **TripSearchUser**: 行程搜索测试
  - 基础搜索功能
  - 带时间条件搜索
  - 带过滤条件搜索
  - 分页搜索测试

- **TripPublishUser**: 行程发布测试
  - 司机发布行程
  - 查看我的行程
  - 更新行程信息
  - 取消行程

- **TripDetailUser**: 行程详情测试
  - 查看行程详情
  - 查看预订情况

- **TripStressUser**: 行程压力测试
  - 快速搜索测试
  - 响应时间监控

### 3. 预订模块测试 (booking_scenario.py)
- **BookingUser**: 预订功能测试
  - 创建预订
  - 查看我的预订
  - 查看预订详情
  - 取消预订

- **DriverBookingUser**: 司机预订管理
  - 查看行程预订
  - 确认预订
  - 拒绝预订

- **BookingStressUser**: 预订压力测试
  - 快速预订操作
  - 并发预订测试

- **ConcurrentBookingUser**: 并发预订测试
  - 多用户抢订同一行程
  - 座位竞争测试

## 🎮 用户角色模拟

### 乘客用户 (PassengerUser) - 权重: 3
**主要行为**:
- 🔍 搜索行程 (权重: 5) - 最频繁的操作

- 📋 查看我的预订 (权重: 2)
- ❌ 取消预订 (权重: 1)

### 司机用户 (DriverUser) - 权重: 1
**主要行为**:
- 📢 发布行程 (权重: 4) - 核心功能
- 📋 查看我的行程 (权重: 3)
- 👥 管理预订 (权重: 2)
- ✅ 确认预订 (权重: 2)
- ❌ 取消行程 (权重: 1)

### 管理员用户 (AdminUser) - 权重: 0.1
**主要行为**:
- 👥 查看所有用户 (权重: 3)
- 🚗 查看所有行程 (权重: 2)
- 📝 查看所有预订 (权重: 2)
- 📞 查看投诉 (权重: 1)

## 📊 性能指标监控

### 关键性能指标 (KPI)
- **响应时间**: 平均 < 500ms, 95% < 2000ms
- **错误率**: < 5%
- **吞吐量**: > 10 RPS
- **并发用户**: 支持100+并发

### 监控维度
- **API级别**: 每个接口的性能表现
- **业务级别**: 完整业务流程的性能
- **系统级别**: 整体系统负载能力
- **用户级别**: 不同用户角色的体验

## 🚀 快速使用指南

### 1. 环境准备
```bash
# 1. 确保Python 3.7+
python --version

# 2. 安装依赖
pip install -r requirements.txt

# 3. 验证环境
python test_setup.py
```

### 2. 启动测试
```bash
# 图形化界面 (推荐新手)
python quick_start.py

# 命令行界面 (推荐高级用户)
python run_tests.py --scenario normal

# Web UI界面 (推荐调试)
python run_tests.py --web-ui
```

### 3. 查看报告
- **HTML报告**: `reports/report_*.html` - 在浏览器中查看
- **CSV数据**: `reports/stats_*.csv` - 用Excel分析
- **JSON数据**: `reports/report_*.json` - 程序化处理

## 🔧 自定义配置

### 修改测试参数
编辑 `config.py` 文件:
```python
# 修改目标服务器
HOST = "http://your-server:8888"

# 修改测试用户
TEST_USERS = [
    {"phone": "your_phone", "password": "your_password", "userType": "PASSENGER"}
]

# 修改性能阈值
PERFORMANCE_THRESHOLDS = {
    "response_time_avg": 300,  # 平均响应时间
    "error_rate": 0.02,        # 错误率 2%
}
```

### 添加新的测试场景
1. 在 `scenarios/` 目录创建新文件
2. 继承 `HttpUser` 类
3. 使用 `@task` 装饰器定义测试任务
4. 在 `run_tests.py` 中添加运行选项

### 关键指标解读
- **绿色指标**: 性能良好，符合预期

## 🔍 故障排除

### 常见问题
1. **连接失败**: 检查后端服务是否启动
2. **认证失败**: 确认测试用户账号密码
3. **依赖错误**: 运行 `pip install -r requirements.txt`
4. **内存不足**: 减少并发用户数

### 调试技巧
```bash
# 详细日志
python run_tests.py --loglevel DEBUG

# 单用户测试
python run_tests.py --users 1 --run-time 30s

# 环境检查
python test_setup.py
```
